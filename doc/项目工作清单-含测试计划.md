## 五、测试目标

测试工作的目标是验证系统是否满足以下要求：

1. 三类传感器能够稳定采集并返回有效数据。
2. 主控能够通过 RS485/Modbus 正确读取三个从机。
3. 告警逻辑能够根据阈值正确切换状态。
4. LED、蜂鸣器和风扇能够按照告警等级执行动作。
5. MQTT 上报数据格式正确，云端或后端能够识别。
6. Web 页面能够显示最新设备状态。
7. 异常场景下系统不会崩溃，本地告警能力不依赖网络。

## 九、验收标准

项目最终验收应满足以下标准：

1. 三个从机能够分别完成光照、温湿度和烟雾数据采集。
2. 主控能够稳定轮询三个从机并获得正确数据。
3. 本地告警等级判断正确，LED、蜂鸣器和风扇动作符合设计。
4. MQTT 上报数据字段完整，云端或后端能够识别并展示。
5. Web 页面能够查看设备最新状态。
6. 文档包含需求分析、详细设计、工作清单、分工计划和测试计划。

本工作清单覆盖了从需求分析、系统设计、嵌入式开发、上云联调、后端开发、测试验证到文档整理的完整过程。任务分配以刘磊负责嵌入式和硬件联调、余新洁负责需求后端和文档测试为主，同时共同参与系统联调。测试计划覆盖传感器、Modbus 通信、告警控制、MQTT 上云、后端页面和异常恢复，能够较完整地验证项目功能和稳定性。
